# real-time-forum

Single-page форум на Go с регистрацией, постами, комментариями, реакциями, уведомлениями и личными сообщениями в реальном времени (WebSocket).

## Возможности

- Регистрация и логин (по `nickname` или `email`), сессии на cookie.
- Лента постов с пагинацией и фильтрами:
  - все посты;
  - мои посты;
  - понравившиеся;
  - по одной или нескольким категориям.
- Создание постов и комментариев.
- Лайки/дизлайки для постов и комментариев.
- Удаление своих комментариев.
- Личные сообщения:
  - список диалогов с превью последнего сообщения;
  - онлайн-статус пользователей;
  - загрузка истории частями (по 10 сообщений).
- Уведомления (новые комментарии/реакции) + отметка как прочитанных.
- Загрузка аватара пользователя (jpg/png/gif/webp, до 2MB).

## Стек

- **Backend:** Go (`net/http`, `database/sql`)
- **WebSocket:** `github.com/gorilla/websocket`
- **База данных:** SQLite (`github.com/mattn/go-sqlite3`)
- **Аутентификация:** cookie-сессии + `bcrypt`
- **Frontend:** Vanilla JS + HTML + CSS (без фреймворков)

## Требования

- Go `1.24+` (в `go.mod` указано `1.24.2`)
- C-компилятор для сборки `go-sqlite3`:
  - macOS: `xcode-select --install`
  - Linux: `build-essential` / `gcc`

## Быстрый старт

```bash
go run .
```

После запуска приложение доступно по адресу: [http://localhost:8080](http://localhost:8080)

### Что создается автоматически

- `forum.db` — SQLite база в корне проекта.
- `static/uploads/avatars/` — директория для аватаров (при первой загрузке файла).

## Структура проекта

```text
.

├── handlers/               # HTTP API и вспомогательные обработчики
```

## Основные API-эндпоинты

### Auth и профиль

- `GET /api/me` — текущий пользователь.
- `POST /api/register` — регистрация.
- `POST /api/login` — вход.
- `POST /api/logout` — выход.
- `POST /api/profile/avatar` — обновить аватар.

### Контент

- `GET /api/categories` — список категорий.
- `GET /api/posts` — лента постов (пагинация/фильтры).
- `GET /api/post/:id` — пост + комментарии.
- `POST /api/post` — создать пост.
- `POST /api/comment` — добавить комментарий.
- `POST /api/delete_comment` — удалить свой комментарий.
- `POST /api/like_post` — лайк/дизлайк поста.
- `POST /api/like_comment` — лайк/дизлайк комментария.

### Личные сообщения и уведомления

- `GET /api/messages/conversations` — список диалогов.
- `GET /api/messages/with/:id` — сообщения с пользователем.
- `GET /api/notifications` — список уведомлений.
- `POST|PATCH /api/notifications/read` — отметить одно уведомление прочитанным.
- `POST|PATCH /api/notifications/read-all` — отметить все прочитанными.
- `GET /ws` — WebSocket канал для real-time событий.

## Маршруты SPA

Обслуживаются через единый `templates/index.html`:

- `/`
- `/login`
- `/register`
- `/create`
- `/feed`
- `/feed/:categoryId`
- `/post/:id`

## Примечания для разработки

- Схема БД и первичные категории инициализируются в `models/database.go`.
- При изменении модели данных удобнее начать с чистой БД:

```bash
rm -f forum.db
go run .
```

- Для проверки real-time сообщений откройте сайт в двух разных браузерах/профилях и войдите под разными пользователями.
